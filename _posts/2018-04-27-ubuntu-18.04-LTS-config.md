# 开启TCP BBR拥塞控制算法

Linux kernel 4.9+ 已经支持`tcp_bbr`，ubuntu 18.04 默认使用4.15内核，可以直接开启

写入配置文件

```sh
sudo bash -c 'cat >> /etc/sysctl.conf' << EOF

net.core.default_qdisc=fq
net.ipv4.tcp_congestion_control=bbr
EOF
```

让内核配置生效

```sh
sudo sysctl -p
```

以下两条命令的结果中都有bbr，说明内核已经开启bbr

```sh
sysctl net.ipv4.tcp_available_congestion_control
sysctl net.ipv4.tcp_congestion_control
```

以下命令看到有 `tcp_bbr`，说明bbr已启动

```sh
lsmod | grep bbr
```

我这边网络环境比较恶劣，开启bbr后大幅改善响应速度

# 全局智能代理

流量分流方案: redsocks2 + iptables白名单 + shadowsocks-libev

redsocks2是一个能重定向TCP/UDP流量的软件，配合iptables规则，只把国外流量转发给redsocks2本地监听端口，利用redsocks2的autoproxy功能，可实现全局智能代理。国内IP直连，国外IP先尝试直连，直连不了则走proxy并缓存IP地址，下次访问此IP直接走proxy，不需要手动维护bloked列表，最大限度节省SS流量

DNS分流方案: overture

overture是一个DNS分流净化软件，可配置多个上游DNS，自动选择最快应答，支持缓存，支持ECS，支持本地Host配置

处理逻辑: 配置主从DNS集合，主DNS集为国内常用DNS服务提供商，DNSPod、AliDNS、114之流，从DNS即备用DNS，一般使用OpenDNS的443端口进行tcp查询，主要用于查询国外域名。用户请求的域名如果在DomainFile中匹配，则直接使用备用DNS解析。若主DNS解析得出的IP不在IPNetworkFile中，则使用备用DNS解析

## 编译redsocks2

```sh
git clone https://github.com/semigodking/redsocks.git

cd redsocks
```

编译前先安装依赖

1. C编译器 (build-essential)
2. libevent2
3. OpenSSL / PolarSSL

一些较新的发行版默认已经安装OpenSSL1.1，并且高版本libevent2，如libevent-2.1.6依赖于OpenSSL1.1，如果系统没有安装openssl，可自行安装openssl 或 libssl-dev

redsocks2作者新增了支持OpenSSL1.1的补丁，不过会导致无法开启自带的shadowsocks功能，好在我们可以自己安装shadowsocks，不影响使用

- 如果系统的OpenSSL是1.0版本，可以按如下命令安装依赖，然后编译

```sh
sudo apt-get install build-essential libevent-dev

make
```

- 如果系统的OpenSSL是1.1版本，可以按如下命令安装依赖，应用补丁，然后编译

```sh
sudo apt-get install build-essential libevent-dev

git apply patches/disable-ss.patch

make
```

- 如果使用PolarSSL编译，可以按如下命令安装依赖，然后编译

```sh
sudo apt-get install build-essential libevent-dev libpolarssl-dev

make USE_CRYPTO_POLARSSL=true
```

编译完成会生成redsocks2可执行文件

新建执行redsocks2的特殊用户

之所以不用当前用户执行，是因为后面iptables的规则需要过滤掉redsocks2本身外发的流量，iptables的owner模块只提供了--uid-owner而没有--pid-owner，只能以执行的用户来过滤，不能以进程过滤

```
iptables -t nat -I SHADOWSOCKS -p tcp -m owner --uid-owner reds2 -j RETURN
```

新建用户reds2

```sh
sudo adduser --shell /bin/false --no-create-home reds2
```

启动命令

```sh
sudo -u reds2 ./redsocks2 -c redsocks.conf
```

redsocks.conf 配置文件

```
base {
    log_debug = off;
    log_info = off;
    //log = stderr;
    log = "file:/path/to/file/redsocks.log";
    //log = "syslog:local7";

    redirector = iptables;
}

redsocks {
    local_ip = 127.0.0.1;
    local_port = 12345;
    ip = 127.0.0.1;
    port = 1080;
    type = socks5;
    //ip = <ss server ip>;
    //port = 8989;
    //type = shadowsocks;
    autoproxy = 1;
    timeout = 3;
    //login = "aes-256-gcm";
    //password = "xxxxxxxxxx";
}

autoproxy {
    no_quick_check_seconds = 3600;
    quick_connect_timeout = 1;
}

ipcache {
    cache_size = 4;
    cache_file = "/path/to/file/ipcache.txt";
    stale_time = 7200;
    autosave_interval = 5;
    port_check = 1;
}
```

## 设置iptables规则

china-ip-list来源，APNIC或IPIP.NET，任选其一

新建脚本download-china-ip-list-APNIC.sh，用于从APNIC下载并过滤出中国地址

```sh
#!/bin/sh

# -c 断点续传 -N 比较服务器文件时间戳
wget -c -N http://ftp.apnic.net/stats/apnic/delegated-apnic-latest

# 筛选
cat delegated-apnic-latest | awk -F '|' '/CN/&&/ipv4/ {print $4 "/" 32-log($5)/log(2)}' | cat > ./china_ip_list.txt
```

新建脚本download-china-ip-list-IPIP.NET.sh，用于从IPIP.NET下载中国地址列表

```sh
#!/bin/sh

wget https://raw.githubusercontent.com/17mon/china_ip_list/master/china_ip_list.txt
```

安装ipset

```sh
sudo apt-get install ipset
```

新建脚本ipset-save.sh，将china-ip-list.txt转换成适用于ipset恢复的格式

单独把ipset add独立出一个文件执行，可以大大减少iptables设置的时间

```sh
#!/bin/sh

# Setup the ipset

ipset -N chnroute hash:net maxelem 65536

for ip in $(cat './china_ip_list.txt'); do
    ipset add chnroute $ip
done

# Save ipset chnroute to file

ipset save chnroute -f ./china_rules.txt

ipset destroy chnroute
```

新建iptables-set.sh

```sh
#!/bin/sh

SOCKS_SERVER=<ss server ip> # SOCKS 服务器的 IP 地址

# Restore the ipset
ipset restore -f ./iptables_rules/china_rules.txt

# 在nat表中新增一个链SHADOWSOCKS
iptables -t nat -N SHADOWSOCKS

################ 不转发部分(保留地址、ss服务器、redsocks2) ################

# Allow connection to the server
iptables -t nat -A SHADOWSOCKS -d $SOCKS_SERVER -j RETURN

# Allow connection to reserved networks
iptables -t nat -A SHADOWSOCKS -d 0.0.0.0/8 -j RETURN
iptables -t nat -A SHADOWSOCKS -d 10.0.0.0/8 -j RETURN
iptables -t nat -A SHADOWSOCKS -d 127.0.0.0/8 -j RETURN
iptables -t nat -A SHADOWSOCKS -d 169.254.0.0/16 -j RETURN
iptables -t nat -A SHADOWSOCKS -d 172.16.0.0/12 -j RETURN
iptables -t nat -A SHADOWSOCKS -d 192.168.0.0/16 -j RETURN
iptables -t nat -A SHADOWSOCKS -d 224.0.0.0/4 -j RETURN
iptables -t nat -A SHADOWSOCKS -d 240.0.0.0/4 -j RETURN

# 不转发redsocks2本身的流量
iptables -t nat -I SHADOWSOCKS -p tcp -m owner --uid-owner reds2 -j RETURN

################ 国内部分 ################

# Allow connection to chinese IPs
iptables -t nat -A SHADOWSOCKS -p tcp -m set --match-set chnroute dst -j RETURN

# 如果你想对 icmp 协议也实现智能分流，可以加上下面这一条
# iptables -t nat -A SHADOWSOCKS -p icmp -m set --match-set chnroute dst -j RETURN

################ 国外部分 ################

# 上面已经用白名单把国内地址过滤掉不处理，国外的部分转发给redsocks2处理

# 当然国外的也可以直接发给Shadowsocks的端口，之所以不直接转发给Shadowsocks
# 是因为国外的并不是都无法访问，只有部分blocked，所以才需要redsocks2的autoproxy，节省ss流量

# tcp流量转发到redsocks2监听端口
iptables -t nat -A SHADOWSOCKS -p tcp -j REDIRECT --to-ports 12345

# Redirect to Shadowsocks
#iptables -t nat -A SHADOWSOCKS -p tcp -j REDIRECT --to-port 1080
# 如果你想对 icmp 协议也实现智能分流，可以加上下面这一条
# iptables -t nat -A SHADOWSOCKS -p icmp -j REDIRECT --to-port 1080

################ 追加规则 ################

# 将SHADOWSOCKS链中所有的规则追加到OUTPUT链中
iptables -t nat -A OUTPUT -p tcp -j SHADOWSOCKS
# 如果你想对 icmp 协议也实现智能分流，可以加上下面这一条
# iptables -t nat -A OUTPUT -p icmp -j SHADOWSOCKS
```

新建iptables-flush.sh，用于停止服务时取消iptables的规则

```sh
#!/bin/sh

iptables -t nat -D OUTPUT -p tcp -j SHADOWSOCKS
# iptables -t nat -D OUTPUT -p icmp -j SHADOWSOCKS

# -F 清空SHADOWSOCKS链中的规则
iptables -t nat -F SHADOWSOCKS

# -X 删除SHADOWSOCKS链
iptables -t nat -X SHADOWSOCKS

ipset destroy chnroute
```

```sh
sudo apt-get install shadowsocks-libev
```

```sh
# & 后台运行作业，使得终端不被占据;

# 2>&1 将标准错误重定向到标准输出
# -u 开启udp转发

#nohup ss-redir -c ss.json -u > ss-redir.log 2>&1 &
```

新建install.sh脚本，用于安装overture

```sh
#!/bin/sh

# 复制程序到/usr/bin下
cp ./overture-linux-amd64 /usr/bin/overture

# 复制配置文件到/etc/overture下
rsync -r --delete --exclude="overture" ./ /etc/overture

# 配置overture服务
bash -c 'cat > /lib/systemd/system/overture.service' << EOF
[Unit]
Description=overture service

[Service]
ExecStart=/usr/bin/overture -c /etc/overture/config.json -l /etc/overture/overture.log
Restart=on-failure
RestartSec=42s

[Install]
WantedBy=multi-user.target
EOF

# 重载修改过的配置文件
systemctl daemon-reload
```

以sudo执行


Ubuntu 18.04 LTS 默认使用systemd-resolved进行dns解析，停掉该服务，再启动overture

停止systemd-resolved

```sh
sudo systemctl stop systemd-resolved
sudo systemctl disable systemd-resolved
```

更改resolv.conf符号连接

```sh
sudo mv /etc/resolv.conf /etc/resolv.conf.bak
sudo ln -s /var/run/NetworkManager/resolv.conf /etc/resolv.conf
```

修改网络连接中的DNS服务器为127.0.0.53

一些软件依赖于NetworkManage或/etc/resolv.conf中的域名服务器才能正常工作，所以我们要把所有DNS的请求接口修改为127.0.0.53:53

重启NetworkManager

```sh
sudo systemctl restart NetworkManager
```

启动overture前，检查53端口是否被占用

```sh
sudo lsof -i:53
```

如果被占用，就杀掉进程(一般被dnsmasq之类的DNS服务占用)

```sh
sudo kill -9 <PID>
```

激活开机启动

```sh
sudo systemctl enable overture
```

启动overture服务

```sh
sudo systemctl start overture
```

检查运行状态

```sh
sudo systemctl status overture
```

```sh
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
```

添加REPO并更新

```sh
echo "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list
```

或者直接追加进sources.list中

```sh
sudo add-apt-repository \
"deb [arch=amd64] https://download.docker.com/linux/ubuntu \
$(lsb_release -cs) \
stable"
```

```sh
sudo apt-get update
```
